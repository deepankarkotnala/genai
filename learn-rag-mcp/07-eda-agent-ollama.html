Part 07 — Capstone Build · 100% Local & Free

📊 Build an EDA Agent with Ollama

Point this agent at any CSV file and ask questions in plain English. It will decide which analyses to run, compute real statistics, and even generate plots — powered by a large language model running entirely on your own machine. No API key. No cost. No data leaving your computer.

What is "EDA"? Exploratory Data Analysis is the first thing a data scientist does with a new dataset: look at its shape, summary statistics, distributions, missing values, and correlations to understand what's there before modeling. We're going to build an agent that does this conversationally — you ask, it explores.
This page ties the whole guide together. We use the LLM concepts from Part 01, the agent loop from Part 03 (think → act → observe), and the tool pattern throughout — but with a local model via Ollama instead of a cloud API. If you skipped Part 03, skim it first; this is that loop applied to data.

1. What we're building (and why local)

A command-line assistant where you load data.csv and have a conversation like:

❓ how many rows are there and what columns?
❓ what's the average age, and are there any missing values?
❓ which two numeric columns are most correlated?
❓ plot a histogram of the price column

Behind the scenes, a local LLM reads your question, picks the right tool (e.g. describe_data, correlation, plot_histogram), your Python code runs it on a real pandas DataFrame, and the model explains the result in plain English.

2. How an EDA agent works

It's the agent loop from Part 03, specialized for a dataset. The DataFrame stays in your program; the LLM never sees the raw data — only the results of the tools it calls. This keeps things fast, private, and accurate (the model does reasoning, pandas does the math).

3. Step 0 — Install & run Ollama

Ollama is a free app that runs open LLMs locally with a simple API. Install it, then pull a model that supports tool calling (essential for an agent).

terminal# 1. Download & install Ollama from https://ollama.com  (Windows/Mac/Linux)

# 2. Pull a tool-capable model (~4.7 GB). Llama 3.1 supports tool calling.
ollama pull llama3.1

# 3. Verify it runs
ollama run llama3.1 "Say hello in one short sentence."

# Ollama also serves an HTTP API at http://localhost:11434 automatically.
Model choice matters for agents. The agent needs a model trained for function/tool calling. Good local options: llama3.1 (8B, solid default), qwen2.5 (great at tools), or mistral-nemo. Tiny models (<3B) often call tools unreliably. If you have a weak machine, try qwen2.5:3b but expect occasional misfires.
Hardware note. An 8B model needs roughly 8 GB of RAM/VRAM to run comfortably. It works on CPU (slower) and is much faster with a GPU or Apple Silicon. The first response is slow (model loads into memory); subsequent ones are quicker.

5. Step 2 — Define the EDA tools

Tools are just normal Python functions that operate on the DataFrame and return a string (the model reads text). Keep outputs concise — the model only needs the result, not megabytes of raw rows.

STEP 2 · TOOLS 
import pandas as pd
import matplotlib
matplotlib.use("Agg")           # save plots to file, no popup window
import matplotlib.pyplot as plt

df = pd.read_csv("data.csv")   # the dataset lives here, in our code

def get_overview():
    """Rows, columns, and data types of the dataset."""
    return (f"Shape: {df.shape[0]} rows x {df.shape[1]} columns.\n"
            f"Columns & types:\n{df.dtypes.to_string()}")

def describe_data():
    """Summary statistics (mean, std, min, max, quartiles) for numeric columns."""
    return df.describe().to_string()

def missing_values():
    """Count of missing (null) values per column."""
    miss = df.isnull().sum()
    return miss[miss > 0].to_string() or "No missing values."

def value_counts(column: str):
    """Frequency of each value in a (usually categorical) column."""
    if column not in df.columns:
        return f"No column '{column}'. Available: {list(df.columns)}"
    return df[column].value_counts().to_string()

def correlation():
    """Correlation matrix between numeric columns (-1 to 1)."""
    num = df.select_dtypes("number")
    if num.shape[1] < 2:
        return "Need at least 2 numeric columns."
    return num.corr().round(2).to_string()

def column_stat(column: str, stat: str):
    """One statistic (mean, median, min, max, sum, std) for one numeric column."""
    if column not in df.columns:
        return f"No column '{column}'."
    s = df[column]
    funcs = {"mean":s.mean, "median":s.median, "min":s.min,
             "max":s.max, "sum":s.sum, "std":s.std}
    if stat not in funcs:
        return f"Unknown stat '{stat}'. Try: {list(funcs)}"
    return f"{stat} of {column} = {funcs[stat]():.2f}"

def plot_histogram(column: str):
    """Save a histogram of a numeric column to a PNG file."""
    if column not in df.columns:
        return f"No column '{column}'."
    plt.figure()
    df[column].hist(bins=10)
    plt.title(f"Histogram of {column}"); plt.xlabel(column); plt.ylabel("count")
    fname = f"hist_{column}.png"
    plt.savefig(fname); plt.close()
    return f"Saved histogram to {fname}"
Design principle. Each tool is small, does one thing, returns a short string, and validates its input (e.g. "no such column"). Good tool design is the difference between an agent that works and one that flails — clear names + docstrings help the LLM choose correctly.

6. Step 3 — Describe the tools to the model

Just like Part 03, we give the LLM a JSON description of each tool so it knows what's available and what arguments to pass. Ollama uses the same OpenAI-style tool schema.

STEP 3 · SCHEMA 
# Map tool names -> the actual Python functions
TOOL_FUNCS = {
    "get_overview": get_overview, "describe_data": describe_data,
    "missing_values": missing_values, "value_counts": value_counts,
    "correlation": correlation, "column_stat": column_stat,
    "plot_histogram": plot_histogram,
}

def tool(name, desc, props=None, required=None):
    """Helper to build one tool schema entry."""
    return {"type": "function", "function": {
        "name": name, "description": desc,
        "parameters": {"type":"object",
            "properties": props or {}, "required": required or []}}}

STR = {"type": "string"}
TOOLS = [
  tool("get_overview",  "Rows, columns and data types of the dataset."),
  tool("describe_data", "Summary stats for all numeric columns."),
  tool("missing_values","Count of missing values per column."),
  tool("correlation",   "Correlation matrix of numeric columns."),
  tool("value_counts",  "Frequency of values in a column.",
       {"column": STR}, ["column"]),
  tool("column_stat",   "One statistic for one numeric column.",
       {"column": STR,
        "stat": {"type":"string",
                 "enum":["mean","median","min","max","sum","std"]}},
       ["column","stat"]),
  tool("plot_histogram","Save a histogram PNG of a numeric column.",
       {"column": STR}, ["column"]),
]
Why the enum for stat? Constraining the allowed values ("mean", "median", …) makes the model far more reliable — it can't invent a stat name your code doesn't support. Tightening schemas is one of the easiest ways to improve a local agent.

7. Step 4 — The agent loop with Ollama

This is the same think → act → observe loop from Part 03, now talking to localhost:11434. A system prompt tells the model its job and to always use tools rather than guess.

STEP 4 · LOOP 
import ollama, json

SYSTEM = ("You are a data analysis assistant. Use the provided tools "
          "to inspect the dataset before answering. Never invent numbers; "
          "always call a tool to get real values, then explain the result "
          "clearly and concisely for a non-technical user.")

def ask_agent(question, history):
    history.append({"role": "user", "content": question})

    for _ in range(6):                      # cap tool-call rounds (safety)
        resp = ollama.chat(model="llama3.1",
                           messages=history, tools=TOOLS)
        msg = resp["message"]
        history.append(msg)

        calls = msg.get("tool_calls")
        if not calls:                       # no tool -> final answer
            return msg["content"]

        for call in calls:               # run each requested tool
            name = call["function"]["name"]
            args = call["function"]["arguments"]   # already a dict
            fn = TOOL_FUNCS.get(name)
            result = fn(**args) if fn else f"Unknown tool {name}"
            print(f"   🔧 {name}({args}) ->", result[:60].replace(chr(10),' '), "...")
            history.append({"role": "tool", "name": name,
                            "content": str(result)})
        # loop again so the model can use the tool output

    return "(Stopped after several tool calls.)"
Trace one turn: you ask "average salary?" → the model returns a tool_calls entry for column_stat(column="salary", stat="mean") → your loop runs it → appends {"role":"tool", ...} with the number → calls the model again → this time it has the value and returns plain text: "The average salary is $81,333." That round-trip is the agent at work — identical in shape to the cloud version in Part 03.

8. Step 5 — The complete program

Everything in one file. Save as eda_agent.py, make sure Ollama is running and data.csv exists, then run python eda_agent.py.

eda_agent.py — completeimport sys, ollama
import pandas as pd
import matplotlib
matplotlib.use("Agg")
import matplotlib.pyplot as plt

CSV   = sys.argv[1] if len(sys.argv) > 1 else "data.csv"
MODEL = "llama3.1"
df = pd.read_csv(CSV)

# ---------- TOOLS ----------
def get_overview():
    return f"Shape: {df.shape[0]} rows x {df.shape[1]} cols.\n{df.dtypes.to_string()}"
def describe_data():
    return df.describe().to_string()
def missing_values():
    m = df.isnull().sum(); m = m[m > 0]
    return m.to_string() if len(m) else "No missing values."
def value_counts(column):
    if column not in df.columns: return f"No column '{column}'. Have: {list(df.columns)}"
    return df[column].value_counts().to_string()
def correlation():
    num = df.select_dtypes("number")
    if num.shape[1] < 2: return "Need 2+ numeric columns."
    return num.corr().round(2).to_string()
def column_stat(column, stat):
    if column not in df.columns: return f"No column '{column}'."
    s = df[column]; f = {"mean":s.mean,"median":s.median,"min":s.min,
        "max":s.max,"sum":s.sum,"std":s.std}
    if stat not in f: return f"Unknown stat '{stat}'."
    return f"{stat} of {column} = {f[stat]():.2f}"
def plot_histogram(column):
    if column not in df.columns: return f"No column '{column}'."
    plt.figure(); df[column].hist(bins=10)
    plt.title(f"Histogram of {column}"); plt.xlabel(column); plt.ylabel("count")
    fn = f"hist_{column}.png"; plt.savefig(fn); plt.close()
    return f"Saved histogram to {fn}"

TOOL_FUNCS = {"get_overview":get_overview,"describe_data":describe_data,
  "missing_values":missing_values,"value_counts":value_counts,
  "correlation":correlation,"column_stat":column_stat,
  "plot_histogram":plot_histogram}

# ---------- TOOL SCHEMAS ----------
def t(n,d,p=None,r=None):
    return {"type":"function","function":{"name":n,"description":d,
      "parameters":{"type":"object","properties":p or {},"required":r or []}}}
S = {"type":"string"}
TOOLS = [
  t("get_overview","Rows, columns and data types."),
  t("describe_data","Summary stats for numeric columns."),
  t("missing_values","Missing values per column."),
  t("correlation","Correlation matrix of numeric columns."),
  t("value_counts","Frequency of values in a column.",{"column":S},["column"]),
  t("column_stat","One stat for one numeric column.",
    {"column":S,"stat":{"type":"string",
     "enum":["mean","median","min","max","sum","std"]}},["column","stat"]),
  t("plot_histogram","Save a histogram PNG of a column.",{"column":S},["column"]),
]

SYSTEM = ("You are a data analysis assistant. Use the tools to inspect the "
  "dataset before answering. Never invent numbers; call a tool to get "
  "real values, then explain results clearly for a non-technical user.")

def ask_agent(question, history):
    history.append({"role":"user","content":question})
    for _ in range(6):
        msg = ollama.chat(model=MODEL, messages=history, tools=TOOLS)["message"]
        history.append(msg)
        calls = msg.get("tool_calls")
        if not calls:
            return msg["content"]
        for c in calls:
            name = c["function"]["name"]; args = c["function"]["arguments"]
            fn = TOOL_FUNCS.get(name)
            out = fn(**args) if fn else f"Unknown tool {name}"
            print(f"   🔧 {name}({args})")
            history.append({"role":"tool","name":name,"content":str(out)})
    return "(Stopped after several tool calls.)"

if __name__ == "__main__":
    print(f"📊 EDA Agent ready. Loaded {CSV} ({df.shape[0]}x{df.shape[1]}). Ctrl+C to quit.\n")
    history = [{"role":"system","content":SYSTEM}]
    while True:
        q = input("❓ ")
        ans = ask_agent(q, history)
        print(f"\n💡 {ans}\n")

11. Troubleshooting

SymptomCause & fix
"connection refused" / can't reach OllamaOllama isn't running. Start the desktop app, or run ollama serve in a terminal. Default port is 11434.
Model never calls toolsUsing a model without tool support. Use llama3.1, qwen2.5, or mistral-nemo. Tiny models call tools poorly.
Wrong column names passedMention exact column names in your question, or add a tool that lists columns. The model can only guess from what it's told.
Very slow responsesNormal on CPU, and the first call loads the model into RAM. Use a smaller model (qwen2.5:3b) or a GPU/Apple Silicon machine.
KeyError: 'tool_calls'Some responses have no tool calls — we use msg.get("tool_calls") (not msg["tool_calls"]) to handle that. Make sure you copied that line.
Loops forever / too many callsThe range(6) cap stops runaway loops. Lower it, or strengthen the system prompt to answer after gathering data.
Hallucinated numbersStrengthen the system prompt ("never invent numbers; always call a tool"). Lower-capability models occasionally guess — a better model helps.
