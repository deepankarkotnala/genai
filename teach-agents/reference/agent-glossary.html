Agent Glossary

The canonical vocabulary for this course. Definitions stay tight on purpose. Designed to print on one or two pages.

Core mechanics

Agent
A language model given a goal and tools, then left to work toward that goal on its own — taking actions and reacting to their results until done. Avoid: "an AI that thinks for itself".

Agent loop
The repeating cycle Perceive → Reason → Act → Observe → (repeat). The defining feature of an agent.

Tool
A function the agent can call to affect or read the world (run code, query data, call an API, run a simulation). The agent's capabilities equal the sum of its tools.

Tool calling / function calling
The mechanism by which an LLM outputs a structured request ("call get_data(tag='T101')") that your code executes, returning the result back into the loop.

System prompt
The standing instructions that define the agent's role, goal, and rules. Sets the agent's "job description".

Context / context window
Everything the model can "see" at once: the prompt, the conversation, tool results so far. Finite — a key design constraint.

Multi-agent (the EnM territory)

Orchestration
Coordinating multiple specialized agents so their combined work produces one business outcome. The central idea in EnM Agents.xlsx.

Orchestrator (router / lead agent)
The agent that decides which sub-agent runs when, and passes results between them. The "conductor".

Handoff
One agent passing its output (and control) to another — e.g. EnM's Anomaly Prediction → RCA → Remediation chain.

Workflow vs. agent
A workflow follows predefined steps (deterministic path). An agent decides its own steps dynamically. Many "agents" in industry are really reliable workflows — and that's often a feature, not a flaw.

Industrial / EnM-specific tools-as-agents

First-principles model
A simulation built from physics/chemistry equations (e.g. reaction kinetics), not learned from data. In EnM it's wrapped as a tool an agent calls.

Differential evolution
A population-based optimization algorithm that searches for a global maximum (e.g. best yield). Deterministic math, not an LLM — wrapped as the "Evolution Agent".

FMEA (Failure Mode & Effects Analysis)
A template that catalogs how a system can fail. EnM's Anomaly agent maps predictions onto FMEA failure modes.

RCA (Root Cause Analysis) / FRACAS / Ishikawa
Methods for finding why something failed. Ishikawa = fishbone diagram of causes; FRACAS = a corrective-action loop.

RPN (Risk Priority Number)
Severity × Occurrence × Detection — a score to rank which root cause to tackle first.

Knowledge graph
Data stored as entities + relationships (not rows). EnM's "Contextualization Agent" builds one from SOPs so other agents can traverse it for remediation steps.

RAG (Retrieval-Augmented Generation)
Giving an LLM relevant documents at query time so it answers from your data, not just its training. The practical backbone of every "Contextualization" / doc-Q&A agent.
